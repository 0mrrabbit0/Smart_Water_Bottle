## 一、项目背景与目标

### 1.1 背景

Smart Water Bottle 是一款基于 STM32F103C8T6 的智能水杯硬件系统，已实现以下功能：
- **水温检测** (DS18B20, ±0.5°C)
- **水质检测** (TDS 传感器, 6 参数: TDS/EC/盐度/比重/温度/硬度)
- **水位监测** (M04 非接触式电容传感器, 3 级)
- **电池管理** (锂电池电压监测 + TP4056 充电)
- **密码锁控** (3 位密码序列 + SG90 舵机)
- **蓝牙传输** (HC-05, USART2, 9600bps, 每 2 秒发送遥测数据)
- **OLED 显示** (SSD1306 128×64, 5 种显示模式)

当前硬件通过蓝牙以 `SWB:T=25.3,TDS=150,WL=3,BAT=85,12:34:56\r\n` 格式发送数据。微信小程序将作为移动端配套应用，提供数据可视化、饮水管理、历史记录等增值功能。

### 1.2 目标

| 目标 | 描述 |
|------|------|
| **数据可视化** | 实时展示水温、水质、水位、电量等硬件数据 |
| **饮水管理** | 设定每日饮水目标，追踪饮水量，智能提醒 |
| **历史记录** | 记录饮水历史、水质趋势，生成周/月报告 |
| **设备管理** | 蓝牙配对、密码设置、固件信息查看 |
| **社交激励** | 饮水排行榜、成就系统、分享打卡 |

### 2.3 蓝牙通信适配说明

硬件使用 HC-05 经典蓝牙模块，而微信小程序仅支持 BLE (蓝牙低功耗)。需要以下适配方案之一：

| 方案 | 描述 | 推荐度 |
|------|------|--------|
| **方案 A: 替换为 BLE 模块** | 将 HC-05 替换为 HM-10/CC2541 等 BLE 模块，硬件改动最小（同为 UART 接口） | ⭐⭐⭐⭐⭐ |
| **方案 B: 添加 BLE 桥接模块** | 保留 HC-05，增加一个 BLE 转 UART 桥接模块 | ⭐⭐⭐ |
| **方案 C: 使用 WiFi 模块** | 替换为 ESP32 (同时支持 WiFi + BLE)，通过云端中转 | ⭐⭐⭐⭐ |

**推荐方案 A**：将 HC-05 替换为 BLE 模块（如 JDY-08/HM-10），UART 接口兼容，固件代码无需修改，仅需更换硬件模块。以下设计均基于 BLE 通信。

### 3.3 饮水记录模块

#### 3.3.1 饮水目标

```
推荐饮水量算法:
  基础量 = 体重(kg) × 30 ml
  运动修正 = 运动时长(min) × 5 ml
  季节修正 = 夏季 +300ml / 冬季 -200ml
  每日推荐 = 基础量 + 运动修正 + 季节修正
```

用户可自定义目标 (500ml ~ 5000ml)，默认推荐 2000ml。

#### 3.3.2 自动饮水检测

通过水位变化自动推断饮水量：

```
检测逻辑:
  1. 持续接收水位数据 (0-3 级)
  2. 检测到水位下降 (如 3→2 或 2→1)
  3. 等待水位稳定 (5 秒内无变化)
  4. 计算饮水量 = (前水位 - 后水位) × 单级容量
  5. 弹窗确认: "检测到饮水约 250ml，是否记录？"
  
单级容量映射 (需根据实际杯型标定):
  Level 3→2: ~150ml
  Level 2→1: ~150ml
  Level 1→0: ~100ml
```

#### 3.3.3 饮水提醒

| 提醒方式 | 实现 | 说明 |
|---------|------|------|
| **定时提醒** | 微信订阅消息 | 每隔 N 小时推送提醒 (默认 2 小时) |
| **智能提醒** | 云函数定时触发 | 根据已饮水量动态调整提醒频率 |
| **久未饮水** | 云函数检测 | 超过 3 小时无饮水记录则推送 |

## 五、BLE 蓝牙通信协议

### 5.1 通信规范

| 项目 | 值 |
|------|-----|
| **服务 UUID** | `0000FFE0-0000-1000-8000-00805F9B34FB` (透传服务) |
| **特征 UUID (RX)** | `0000FFE1-0000-1000-8000-00805F9B34FB` (设备→小程序) |
| **特征 UUID (TX)** | `0000FFE2-0000-1000-8000-00805F9B34FB` (小程序→设备) |
| **MTU** | 20 bytes (默认) / 协商最大 512 bytes |
| **编码** | UTF-8 ASCII |

### 5.2 设备上报数据 (设备 → 小程序)

沿用现有硬件协议格式：

```
SWB:T=25.3,TDS=150,WL=3,BAT=85,12:34:56\r\n
```

解析规则：

| 字段 | 含义 | 类型 | 单位 |
|------|------|------|------|
| `T` | 水温 | float | °C |
| `TDS` | 水质 | uint16 | ppm |
| `WL` | 水位 | uint8 | 0-3 级 |
| `BAT` | 电量 | uint8 | 0-100% |
| 最后字段 | 时间 | string | HH:MM:SS |

### 5.3 小程序下发指令 (小程序 → 设备)

定义扩展指令集：

| 指令 | 格式 | 说明 |
|------|------|------|
| **远程开锁** | `CMD:UNLOCK,PWD=375\r\n` | 发送密码开锁 |
| **修改密码** | `CMD:SETPWD,OLD=375,NEW=123\r\n` | 修改锁密码 |
| **同步时间** | `CMD:SYNC,TIME=12:34:56,DATE=26-03-31\r\n` | 同步 RTC |
| **查询状态** | `CMD:QUERY\r\n` | 主动请求数据 |
| **设置上报间隔** | `CMD:INTERVAL,MS=2000\r\n` | 设置数据上报间隔 |

设备响应格式：

```
RSP:OK\r\n          // 指令执行成功
RSP:ERR,CODE=01\r\n // 指令执行失败 (01=密码错误, 02=格式错误)
```

### 5.4 BLE 连接流程

```
小程序                                    BLE 设备
  │                                         │
  │── wx.openBluetoothAdapter() ──>         │  1. 初始化蓝牙适配器
  │                                         │
  │── wx.startBluetoothDevicesDiscovery() ─>│  2. 扫描 (过滤 name="SWB-")
  │                                         │
  │<── onBluetoothDeviceFound ──────────────│  3. 发现设备
  │                                         │
  │── wx.createBLEConnection() ──────────>  │  4. 建立连接
  │                                         │
  │── wx.getBLEDeviceServices() ──────────> │  5. 获取服务列表
  │                                         │
  │── wx.getBLEDeviceCharacteristics() ───> │  6. 获取特征值
  │                                         │
```

## 八、BLE 服务核心实现

### 8.1 BLE 管理器 (services/ble.js)

```javascript
// BLE 服务核心类
class BLEManager {
  constructor() {
    this.deviceId = ''
    this.serviceId = '0000FFE0-0000-1000-8000-00805F9B34FB'
    this.rxCharId = '0000FFE1-0000-1000-8000-00805F9B34FB'
    this.txCharId = '0000FFE2-0000-1000-8000-00805F9B34FB'
    this.connected = false
    this.dataBuffer = ''
    this.listeners = []
  }

  // 扫描设备
  async scan(timeout = 10000) {
    await wx.openBluetoothAdapter()
    return new Promise((resolve, reject) => {
      const devices = []
      wx.onBluetoothDeviceFound((res) => {
        res.devices.forEach(d => {
          if (d.name && d.name.startsWith('SWB-')) {
            devices.push(d)
          }
        })
      })
      wx.startBluetoothDevicesDiscovery({
        allowDuplicatesKey: false
      })
      setTimeout(() => {
        wx.stopBluetoothDevicesDiscovery()
        resolve(devices)
      }, timeout)
    })
  }

  // 连接设备
  async connect(deviceId) {
    this.deviceId = deviceId
    await wx.createBLEConnection({ deviceId })
    const { services } = await wx.getBLEDeviceServices({ deviceId })
    const { characteristics } = await wx.getBLEDeviceCharacteristics({
      deviceId, serviceId: this.serviceId
    })
    // 订阅数据通知
    await wx.notifyBLECharacteristicValueChange({
      deviceId, serviceId: this.serviceId,
      characteristicId: this.rxCharId, state: true
    })
    wx.onBLECharacteristicValueChange((res) => {
      this._onData(res.value)
    })
    this.connected = true
  }

  // 发送指令
  async send(command) {
    const buffer = new ArrayBuffer(command.length)
    const view = new Uint8Array(buffer)
    for (let i = 0; i < command.length; i++) {
      view[i] = command.charCodeAt(i)
    }
    await wx.writeBLECharacteristicValue({
      deviceId: this.deviceId,
      serviceId: this.serviceId,
      characteristicId: this.txCharId,
      value: buffer
    })
  }

  // 接收数据处理 (拼包 + 解析)
  _onData(arrayBuffer) {
    const str = String.fromCharCode.apply(null, new Uint8Array(arrayBuffer))
    this.dataBuffer += str
    while (this.dataBuffer.includes('\r\n')) {
      const idx = this.dataBuffer.indexOf('\r\n')
      const line = this.dataBuffer.substring(0, idx)
      this.dataBuffer = this.dataBuffer.substring(idx + 2)
      if (line.startsWith('SWB:')) {
        const data = this._parseSWB(line)
        this.listeners.forEach(fn => fn(data))
      }
    }
  }

  // 解析 SWB 协议
  _parseSWB(line) {
    // "SWB:T=25.3,TDS=150,WL=3,BAT=85,12:34:56"
    const body = line.substring(4)
    const parts = body.split(',')
    const data = {}
    parts.forEach(part => {
      if (part.includes('=')) {
        const [key, val] = part.split('=')
        data[key] = isNaN(val) ? val : Number(val)
      } else if (part.includes(':')) {
        data.time = part
      }
    })
    return {
      temperature: data.T,
      tds: data.TDS,
      waterLevel: data.WL,
      battery: data.BAT,
      time: data.time,
      timestamp: Date.now()
    }
  }

  // 注册数据监听
  onData(callback) {
    this.listeners.push(callback)
  }

  // 断开连接
  async disconnect() {
    if (this.deviceId) {
      await wx.closeBLEConnection({ deviceId: this.deviceId })
      this.connected = false
    }
  }
}

module.exports = new BLEManager()
```

## 十一、性能与安全

### 11.1 性能优化

| 策略 | 说明 |
|------|------|
| **本地缓存** | 使用 `wx.setStorageSync` 缓存最新设备数据，离线可看 |
| **分页加载** | 饮水记录列表分页 (每页 20 条) |
| **数据聚合** | 统计数据在云端预聚合 (daily_summary)，减少客户端计算 |
| **图表懒加载** | 图表组件按需加载，非首屏不初始化 |
| **BLE 数据节流** | 2 秒接收周期已足够，UI 更新限制 1 次/秒 |
| **图片压缩** | 所有静态资源 CDN + WebP 格式 |

### 11.2 安全设计

| 安全项 | 措施 |
|--------|------|
| **身份认证** | 微信 openid 天然鉴权，无需额外登录 |
| **数据权限** | 云数据库安全规则：用户只能读写自己的数据 |
| **BLE 安全** | 设备绑定后记录 MAC，防止他人连接 |
| **密码传输** | BLE 近场通信，明文可接受；如需加密可增加简单异或 |
| **输入校验** | 云函数对所有入参做类型和范围校验 |
| **频率限制** | 云函数调用频率限制，防止刷量 |

### 11.3 异常处理

| 场景 | 处理策略 |
|------|---------|
| **BLE 断连** | 自动重连 (最多 3 次，间隔 2/4/8 秒)，失败提示用户 |
| **数据解析失败** | 丢弃异常帧，记录日志，不影响后续数据 |
| **云函数超时** | 重试 1 次，失败后本地缓存待上传 |
| **水位跳变** | 过滤异常跳变 (如 0→3→0)，防止误报饮水 |
| **低电量** | 电量 <20% 显示警告，<10% 推送通知 |
